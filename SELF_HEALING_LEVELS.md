# SELF_HEALING_LEVELS.md

## 修复级别定义

### 概述

自愈系统定义四个修复级别（L0-L3），从纯观察到受限自动合入，逐级提升自动化程度。

---

## L0: 观察 (Observe)

### 描述
仅记录失败事件，生成报告，人工审查后决定是否修复。

### 行为
- ✅ 接收失败事件
- ✅ 生成 failure bundle
- ✅ 记录到 task ledger
- ✅ 发送通知/报警
- ❌ 不执行任何自动修复

### 适用场景
- 首次出现的未知问题
- 高风险操作失败
- 触及边界约束的失败
- 系统初始化阶段

### 人工介入
必须人工审查后，手动选择修复方案。

---

## L1: 沙箱修复 (Sandbox Repair)

### 描述
在隔离环境（临时 worktree/分支）执行修复，验证通过后产出报告，等待人工确认。

### 行为
- ✅ 接收失败事件
- ✅ 生成 failure bundle
- ✅ 诊断归因（L1分类 + L2签名）
- ✅ 生成候选修复方案
- ✅ 在沙箱执行修复
- ✅ 运行 Gate A/B/C
- ✅ 产出 patch + validation report
- ❌ 不修改主分支

### 沙箱环境
```
temp-worktree/
├── .git/               # 独立 git 上下文
├── src/                # 复制的源代码
├── tests/              # 测试套件
└── reports/            # 验证报告
```

### 适用场景
- 已知问题类型的重复出现
- 低风险文件修改
- 有历史成功案例的问题

### 人工介入
审查 validation report 后，手动应用 patch。

---

## L2: 修复分支提交 (Branch Commit)

### 描述
自动创建修复分支，提交修复 commit，推送分支，产出 PR/报告，等待人工合并。

### 行为
- ✅ 接收失败事件
- ✅ 完整诊断流程
- ✅ 沙箱验证通过
- ✅ 自动创建分支 `auto-fix/<signature>-<timestamp>`
- ✅ 提交修复 commit
- ✅ 推送分支到远程
- ✅ 生成 PR 描述
- ❌ 不自动合并到主分支

### 分支命名
```
auto-fix/edit-failed-missing-file-20260314-233918
auto-fix/tool-failed-path-error-20260314-234015
```

### Commit 格式
```
[AutoFix] <问题签名>

- 问题: <简要描述>
- 根因: <诊断结果>
- 修复: <方案描述>
- 验证: <Gate 结果>

Auto-generated by Self-Healing System
Signature: <signature>
Confidence: <score>
```

### 适用场景
- 高置信度的已知问题
- 有完善回归测试的问题
- 团队审查流程完善

### 人工介入
审查 PR，确认无误后手动合并。

---

## L3: 受限自动合入 (Auto Merge)

### 描述
通过所有验证后，自动合并到主分支。需要严格的前提条件。

### 行为
- ✅ 接收失败事件
- ✅ 完整诊断流程
- ✅ 沙箱验证通过
- ✅ 创建分支并提交
- ✅ 通过 Gate A/B/C
- ✅ **自动合并到主分支**
- ✅ 记录到变更日志

### 前提条件（必须全部满足）

| 条件 | 说明 |
|-----|------|
| 问题签名命中历史成功案例 | 该 signature 有 ≥3 次成功修复记录 |
| 成功率 ≥ 90% | 该 signature 的历史成功率 |
| 变更半径 ≤ 5 行 | 修改范围足够小 |
| 所有 Gate 通过 | Gate A/B/C 全部通过 |
| 无 negative check 失败 | 回归测试全部通过 |
| 非禁区文件 | 不在 SELF_HEALING_BOUNDARIES.md 列表中 |
| 非核心时段 | 避开业务高峰期 |

### 适用场景
- 极高置信度的重复问题
- 紧急修复需求
- 无人值守运维

### 人工介入
事后审查变更日志，如有问题执行回滚。

---

## 级别升级条件

### L0 → L1
- 问题 signature 被识别
- 有至少 1 个历史成功案例
- 团队批准升级

### L1 → L2
- 该 signature 沙箱验证成功率 ≥ 80%
- 有完善的回归测试
- 团队批准升级

### L2 → L3
- 该 signature 分支验证成功率 ≥ 90%
- 连续 10 次无人工干预成功
- 管理层批准升级

---

## 级别降级触发

以下情况自动降级：

| 触发条件 | 降级动作 |
|---------|---------|
| 自动修复引入新问题 | L3→L2 或 L2→L1 |
| 连续 3 次验证失败 | 降级一级 |
| 触及禁区文件 | 立即降级至 L0 |
| 安全审计异常 | 立即降级至 L0 |
| 人工投诉 | 降级一级 |

---

## 配置示例

```yaml
# config/healing_levels.yaml

default_level: L1

overrides:
  # 按问题签名配置
  signatures:
    "edit_failed:file_not_found":
      level: L2
      auto_merge_threshold: 0.95
    
    "tool_failed:permission_denied":
      level: L1
      requires_approval: true
    
    "test_failed:assertion_error":
      level: L2
  
  # 按文件路径配置
  paths:
    "src/core/*.py":
      max_level: L1
    
    "docs/*.md":
      max_level: L3
  
  # 按时间段配置
  time_windows:
    "00:00-06:00":
      # 夜间允许更高自动化
      bonus_level: +1
    
    "09:00-18:00":
      # 工作时间更保守
      max_level: L1
```
