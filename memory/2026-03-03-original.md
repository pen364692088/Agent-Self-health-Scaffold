## 关键决策

### 串行子代理执行模式 ✅

**问题**: 子代理回调可能丢失/延迟，导致串行流程中断

**解决方案**:
1. SESSION-STATE.md 记录工作流状态
2. 子代理A完成 → 手动检查状态 → 启动子代理B
3. 全部完成 → 发送最终通知

**模式**:
```yaml
# SESSION-STATE.md
pending_workflow:
  active: true
  type: serial_subagents
  steps:
    - id: A
      runId: "xxx"
      status: running
    - id: B
      runId: null
      status: pending
  notify_on_done: "✅ 完成"
```

**执行流程**:
```
1. spawn 子代理A
2. 更新 SESSION-STATE.md (A: running)
3. 等待回调或主动 poll subagents list
4. A 完成 → 更新 (A: done)
5. spawn 子代理B
6. 更新 SESSION-STATE.md (B: running)
7. B 完成 → 更新 (B: done)
8. 发送最终通知，设置 active: false
```

### 回调问题诊断

**现象**: 收到旧的回调消息（v1测试的结果混入v2测试）

**原因**: 回调系统可能有延迟/重叠

**解决**: 
- 不完全依赖回调
- 主动 poll `subagents list` 检查状态
- SESSION-STATE.md 作为唯一真相源

## MVP-8.2 跨进程 Hash 稳定性修复 (2026-03-03 09:45 CST)

**项目路径**: `/home/moonlight/Project/Github/MyProject/Emotion/OpenEmotion`
**分支**: `feature-emotiond-mvp`
**Commit**: `2abda02`

### 问题：self_hash 跨进程不稳定

**根因**:
```python
"hash_excludes": list(self._HASH_EXCLUDE_TOP | self._HASH_EXCLUDE_NESTED)
```
- set/frozenset 的迭代顺序受 PYTHONHASHSEED 影响
- 不同进程可能有不同迭代顺序
- 导致跨进程 self_hash 不一致

**修复**:
```python
"hash_excludes": sorted(self._HASH_EXCLUDE_TOP | self._HASH_EXCLUDE_NESTED)
```

   - 用 subprocess 启动 4 个独立进程

### 两条产品级加固

#### 1. 公开 API 替代私有字段访问
- **问题**: `narrative_memory._by_target.clear()` 直接访问私有字段
- **风险**: 内部结构变化会导致 verifier 崩溃

**修复**: 添加公开 API
```python
class NarrativeMemory:
    def reset(self, target_id: Optional[str] = None) -> None:
        """Reset narrative memory state."""
        if target_id is None:
            self._by_target.clear()
        elif target_id in self._by_target:
            del self._by_target[target_id]
    
    def get_all_target_ids(self) -> List[str]:
        """Get list of all target_ids with state."""
```

**更新**: 所有测试和 verifier 改用 `narrative_memory.reset()`

#### 2. 跨平台临时目录
- **问题**: `/tmp` 路径在 Windows/某些环境不存在
- **风险**: CI 或跨平台开发会失败

**修复**: 使用标准库 + 环境变量覆盖
```python
import tempfile
TEMP_BASE = Path(os.environ.get("OPENEMOTION_TMP_DIR", tempfile.gettempdir()))
```

**支持**:
- 默认: `tempfile.gettempdir()` (跨平台)
- 覆盖: `OPENEMOTION_TMP_DIR` 环境变量
