## Session: 僵尸任务清理修复 (12:00 CST)

### 问题诊断
用户报告之前的会话"一直在报 subagent 完成了任务，像卡住了一样一直重复"。

**根因分析**：
1. `subtask-orchestrate run` 只注册任务，不执行 spawn/join
2. 任务 deadline 过期后没有自动清理
3. cron 每 2 分钟检查 pending 任务但无法正确处理
4. 僵尸任务积累导致"卡住重复"表象

### 修复内容

**1. orchestrator.py 新增方法**：
- `cleanup_zombie_tasks()` - 清理过期僵尸任务
- `get_stuck_tasks()` - 诊断卡住的任务

**2. subtask-orchestrate 新增命令**：
```bash
subtask-orchestrate stuck          # 查看卡住的任务
subtask-orchestrate gc --zombie    # 清理僵尸任务
subtask-orchestrate list           # 列出 pending 任务
```

**3. callback-handler 新增**：
- `--auto-cleanup` - 自动清理僵尸任务

**4. cron 配置更新**：
- `cc-godmode-continue` 添加 Step 1: 自动清理僵尸任务

**5. SOUL.md 修复**：
- 添加 `.openclaw` 目录写入规则（最高优先级）
- 避免 edit/write 工具失败问题

### 测试验证
| 测试项 | 结果 |
|--------|------|
| 创建僵尸任务 | ✅ |
| stuck 检测 | ✅ |
| gc --zombie 清理 | ✅ |
| callback-handler --auto-cleanup | ✅ |
| cron 配置更新 | ✅ |

### Artifacts
- 修复文件: `tools/orchestrator/orchestrator.py`
- 修复文件: `tools/subtask-orchestrate`
- 修复文件: `tools/callback-handler`
- 配置文件: `~/.openclaw/cron/jobs.json`
- 规则文件: `SOUL.md`

### Lessons Learned
1. **工具设计缺陷**：两步操作（注册 + spawn/join）容易导致不一致
2. **必须有超时清理**：deadline 过期后应自动标记为 timeout
3. **.openclaw 路径规则**：修改此类文件必须用 exec，不能用 edit/write
