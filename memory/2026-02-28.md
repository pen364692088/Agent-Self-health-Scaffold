## MVP-2.1.1 (06:10 CST)

### 目标
防止客户端伪造 `meta.source="system"` 绕过权限门控。

### 实现
1. **Token-based source resolution** (`emotiond/security.py`)
   - `Authorization: Bearer <token>` 或 `X-Emotiond-Token` header
   - `EMOTIOND_SYSTEM_TOKEN` / `EMOTIOND_OPENCLAW_TOKEN` 环境变量
   - 未匹配 → source="user"

2. **Meta sanitization**
   - User source 白名单: `subtype`, `seconds`, `client_source`
   - `source` 由服务器控制，不在白名单中
   - 未知字段 → DENY + 审计记录

3. **time_passed clamp**
   - User source: seconds ∈ [1, 300]
   - 超出 → clamp + `clamped_from` 审计

### 关键学习

#### 1. importlib.reload() 破坏全局状态引用
**问题**: 多个测试文件使用 `importlib.reload()` 重载模块，导致全局状态对象（如 `emotion_state`）在不同测试间变成不同实例。
**表现**: 测试 A 设置 `emotion_state.social_safety = 0.45`，但测试 B 的 `generate_explanation()` 读取的是旧实例的值 0.6。
**解决**: 
- 移除所有 `importlib.reload()` 调用
- 使用 `import emotiond.core as core` 模块引用
- 显式重置全局状态变量

#### 2. daemon_manager 状态残留
**问题**: FastAPI `TestClient(app)` 会触发 `startup_event`，启动 daemon_manager 后台任务，残留到后续测试。
**解决**: 在每个测试 fixture/teardown 中显式重置：
```python
daemon.daemon_manager.running = False
daemon.daemon_manager.loops = {}
```

#### 3. 数据库表需要显式初始化
**问题**: `TestClient(app)` 使用默认数据库路径，新表（如 `decisions`）未创建。
**解决**: 在测试 setup 中显式调用 `db.init_db()`

## MVP-3.1 Target-Specific Prediction Residual Layer (2026-02-28 10:29-10:51 CST)

### 任务目标
将全局预测 `predicted_deltas` 升级为两层结构：
- **Global prior**: 保留现有全局预测
- **Per-target residual**: 每个 target 单独学习偏差/教训

### 核心算法
```
pred_total = pred_global + α(target, action) * pred_residual(target, action)
α = n / (n + k)  # 收缩因子，k=20
```

| **DB** | 新增 `predicted_deltas_target` 表 |
