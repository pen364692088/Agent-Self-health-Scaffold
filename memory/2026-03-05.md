## Session 2: CI Workflow YAML 修复

### Summary
修复 GitHub Actions workflow 文件中的 YAML/shell 语法冲突，导致 GitHub 无法解析 workflow 文件。

### Key Decisions

**问题模式**：
1. `python -c "..."` 多行字符串 → shell 语法错误（`-c` 参数必须单行）
2. `python << 'PY'` heredoc → YAML 缩进冲突（结束标记必须顶格）

**修复方案**：
| Workflow | 方案 |
|----------|------|
| `mvp11-hardening.yml` | 提取到 `scripts/ci/mvp11_print_pointers.py` |
| `mvp9-eval.yml` | 提取到 `scripts/ci/mvp9_analyze_failures.py` + `mvp9_check_threshold.py` |

### 新增文件
```
scripts/ci/
├── mvp11_print_pointers.py
├── mvp9_analyze_failures.py
└── mvp9_check_threshold.py
```
