# Subagent Run - One-Shot Reliable Execution

**The ONLY recommended way to spawn subagents.**

## Why This Exists

Direct `sessions_spawn` can cause tasks to stall when callbacks are lost. This skill wraps spawn with:
- Orchestrator registration
- Join/Poll mechanism
- Mailbox fallback
- Auto-resume of pending tasks

## Quick Start

```bash
subtask-orchestrate run "<task>" -m <model> --timeout 300
```

This outputs:
```
Task ID:    task_abc123
Run ID:     run_xyz
Model:      iflow/qwen3-coder-plus

Step 1: sessions_spawn runtime=subagent model=... task='...'
Step 2: subtask-orchestrate join -t 300 --task-id task_abc123
```

**Both steps are mandatory.**

## When to Use

**Trigger conditions** (use this skill when):
- You need to spawn a subagent for any task
- You have long-running parallel tasks
- You need reliable completion tracking

**Do NOT use** (use this skill instead):
- ❌ `sessions_spawn runtime=subagent ...` (direct call)
- ❌ Manual subagent management

## Enforcement Rules (GATES)

**Gate 1**: Only execute sessions_spawn generated by `subtask-orchestrate run`
**Gate 2**: Always `subtask-orchestrate resume` before starting new tasks
**Gate 3**: Must join after spawn (pending=0 before continuing)
**Gate 4**: Check pending status before final output

## Full Workflow

### Step 1: Check Pending First
```bash
subtask-orchestrate resume -t 60
```

### Step 2: Run One-Shot
```bash
subtask-orchestrate run "Implement feature X" -m iflow/qwen3-coder-plus --timeout 300
```

### Step 3: Execute Spawn Command
```
sessions_spawn runtime=subagent model=iflow/qwen3-coder-plus task='Implement feature X'
```

### Step 4: Join and Collect
```bash
subtask-orchestrate join -t 300 --task-id <task_id>
```

### Step 5: Verify Completion
```bash
subtask-orchestrate list -v
```

## CLI Reference

```bash
# One-shot entry point

subtask-orchestrate join -t <timeout> --task-id <id>

# List tasks
subtask-orchestrate list -v

# Cleanup
```

## Python API

```python
from orchestrator import Orchestrator

orch = Orchestrator()

# One-shot spawn
task_id, run_id, session_key = orch.spawn_task("description", model="...")

# Attach to existing run
task_id = orch.attach_subtask(run_id, session_key, "description")

# Join with dual timeout
orch.join_all(timeout=300, per_task_timeout=600)
```

- Mailbox: `reports/subtasks/<task_id>.done.json`
- Events log: `reports/orchestrator/events.jsonl`

## Enforcer

Run compliance check:
```bash
~/.openclaw/workspace/tools/orchestrator/enforcer
```

Checks:
1. Pending tasks status
2. Spawn via run command (has run_id)
3. Orphan task detection
4. Spawn→join pairing

## Error Recovery

If task stuck in pending:
```bash
subtask-orchestrate resume -t 300
```

If orphan tasks exist:
```bash
subtask-orchestrate gc --days 7
```
