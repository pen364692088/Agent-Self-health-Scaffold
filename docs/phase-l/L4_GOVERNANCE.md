# Phase L-4: 治理与边界补全

**日期**: 2026-03-17

---

## 治理边界定义

### 低风险 Agents (default, healthcheck)
- **权限边界**: 标准 runtime 权限
- **审计要求**: 基础日志记录
- **使用场景**: 通用备选、健康检查
- **治理级别**: 默认

### 中风险 Agents (acp-codex, codex, mvp7-coder)
- **权限边界**: 代码执行权限
- **审计要求**: 完整操作日志
- **使用场景**: ACP harness、编码任务
- **治理级别**: 标准 + 代码审计

### 高风险 Agent (cc-godmode)
- **权限边界**: 高权限操作
- **审计要求**: 全量审计 + 人工确认
- **使用场景**: 架构决策、治理审计、异常复盘
- **治理级别**: 强化

---

## cc-godmode 专属治理边界

### 允许场景
1. 架构决策审计
2. 治理规则审查
3. 异常复盘分析
4. 规则变更评估

### 禁止场景
1. 自动执行敏感操作
2. 修改系统配置
3. 删除数据
4. 外部通信

### 调用要求
1. 必须通过 main agent 发起
2. 必须有明确的审计目的
3. 必须记录完整调用链
4. 结果必须人工确认

### 审计日志
- 调用时间
- 调用者 (main agent)
- 调用目的
- 执行结果
- 人工确认状态

## 配置建议

### 批量处理 (5 agents)
- 添加到 acp.allowedAgents
- 无需额外治理配置
- 通过 ACP harness 调用

### 单独处理 (cc-godmode)
- 添加到 acp.allowedAgents
- 配置专属治理规则
- 定义使用边界
- 设置审计触发器
