# CC_GODMODE_BOUNDARY.md

## Goal
cc-godmode is a governance layer, not a daily workflow crutch.

## Use cc-godmode for
- architecture decisions
- major refactors
- cross-module tradeoffs
- failure review and policy repair
- gate review before important launches

## Do not use cc-godmode for
- normal workflow progression
- inbox checks
- receipt handling
- ordinary retries
- routine subtask spawning
- ordinary tool ordering

## Rule
If something is frequent, easy to forget, and expensive to do wrong, systemize it.
Do not rely on memory or prompt reminders for critical workflow steps.
