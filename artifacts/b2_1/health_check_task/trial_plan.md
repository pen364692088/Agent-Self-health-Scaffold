# B2-1 试运行计划：周期性健康检查任务

## 1. 任务定义

**任务名称**: Agent 健康检查 + 状态写回

**任务描述**: 定期执行 Agent 自身健康检查，生成状态报告并写回 JSONL 历史记录。

## 2. 为什么选择此任务

| 标准 | 满足情况 |
|------|----------|
| 真实业务需求 | ✅ Agent 需要持续监控自身状态 |
| 重复发生 | ✅ 周期性执行，已记录 6 次 |
| 低到中风险 | ✅ 只读检查 + 状态记录 |
| 可验证 | ✅ JSONL 格式可解析验证 |
| 可回放 | ✅ 历史记录可追溯 |
| 失败代价低 | ✅ 检查失败不影响核心功能 |
| 可人工接管 | ✅ 可手动执行检查脚本 |
| 不依赖并发能力 | ✅ 单任务串行 |
| 不依赖复杂编排 | ✅ 单步任务 |
| 不需要复杂 rollback | ✅ 无副作用 |

## 3. 执行流程

```
Step 1: 执行健康检查脚本
Step 2: 解析检查结果
Step 3: 写入历史记录
Step 4: 生成证据文件
Step 5: 验证状态一致性
```

## 4. 试运行目标

- 至少完成 3 次真实运行
- 每次运行生成完整证据
- 验证状态/证据/副作用一致性
- 验证可重复执行

## 5. 验收标准

- [ ] 3 次运行全部成功
- [ ] 每次运行有完整 JSONL 记录
- [ ] 状态文件正确更新
- [ ] 证据链完整
- [ ] 无重复执行问题
- [ ] 无状态不一致

## 6. 交付物清单

- `trial_plan.md` - 本文件
- `run_log.json` - 运行日志
- `run_summary.md` - 运行总结
- `validation_report.md` - 验证报告
- `stability_summary.md` - 稳定性总结
- `evidence/` - 证据目录
- `history/health_history.jsonl` - 历史记录
