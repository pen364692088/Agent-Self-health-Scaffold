# B2-1 受控生产试运行计划

## 任务选择

**任务**: `b21_health_report_daily` - 周期性健康状态报告

**为什么适合**:
1. **真实重复发生**: 可每天执行，生成健康报告
2. **低风险**: 只读系统状态，只写报告文件
3. **可验证**: 报告内容可检查
4. **可人工接管**: 失败可手动生成
5. **有实际价值**: 监控自治系统和仓库健康

## 试运行计划

- 运行次数: 3 次以上
- 运行间隔: 手动触发（模拟周期）
- 观察重点:
  - 任务发现稳定性
  - 步骤执行稳定性
  - 状态/证据一致性
  - 副作用可验证性

## 任务定义

### 步骤

1. S01: 扫描 runtime 目录，统计执行器文件
2. S02: 扫描 artifacts/tasks，统计任务数量和状态
3. S03: 检查 ledger 文件完整性
4. S04: 生成健康报告 JSON
5. S05: 追加到历史记录文件

### 产出

- 健康报告 JSON 文件
- 历史记录追加
- 验证报告

## 验收标准

- [ ] 3 次以上成功运行
- [ ] 每次状态正确
- [ ] 证据完整
- [ ] 副作用一致
- [ ] 历史记录可追溯
