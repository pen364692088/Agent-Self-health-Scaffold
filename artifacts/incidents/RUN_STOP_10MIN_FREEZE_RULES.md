# Phase 0: 冻结错误修复方式

## 生效时间: 2026-03-15 02:33 CDT

## 强制约束

1. **禁止降低 Gate 标准** - 不允许任何 "为了过 Gate 而降低 Gate 要求" 的改动
2. **禁止缩小测试范围** - 不允许把 E2E 改成 smoke test 作为正式修复
3. **禁止无根因修改** - 不允许在未确认根因前直接改多个模块
4. **必须先证据后修复** - 必须先做证据采集，再做根因分类，再做最小修复
5. **结论必须有证据** - 所有结论必须附带日志/退出码/配置证据
6. **timeout 必须明确** - 若是 timeout/lease/watchdog 问题，必须明确指出触发源头和阈值数值

## 违规后果

违反以上约束将触发 policy/meta_rules.yaml 中的违规检测：
- 修改 Gate 定义 -> FAIL
- 将 SKIP 伪装为 PASS -> FAIL
- 缺失证据的修复 -> FAIL
