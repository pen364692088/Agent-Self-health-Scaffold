# B2-3 试运行计划：连续真实日运行观察

## 1. 目标

验证自治系统在连续多日运行中的稳定性，而非单次或短时任务执行。

## 2. 观察周期

| Day | 观察口径日期 | 会话记录日期 | 状态 |
|-----|--------------|--------------|------|

| Day 2 | 2026-03-18 | 2026-03-17 | ⏳ 待执行 |
| Day 3 | 2026-03-19 | 2026-03-18 | ⏳ 待执行 |

**日期口径说明**：
- 观察口径日期：UTC+0，用于文件命名和报告日期
- 会话记录日期：America/Winnipeg (UTC-6)，用户所在时区

## 3. 每日任务

### Task A: 健康检查
- 工具: `agent-health-check`
- 副作用: 更新 `current_health.json`，追加 `health_history.jsonl`
- 验证: 状态 GREEN，记录完整

### Task B: 策略报告
- 工具: `policy-daily-report --save`
- 副作用: 生成 `daily_YYYY-MM-DD.md`
- 验证: 文件存在，格式正确，幂等

## 4. 每日验收清单（固定）

每天必须检查以下 6 项：

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 1 | 健康检查状态 | GREEN |
| 2 | 当日报告生成 | 文件存在且日期正确 |
| 3 | 文件路径与日期一致 | `daily_{观察口径日期}.md` |
| 4 | 内容与真源一致 | 报告数据 = 工具输出 |
| 5 | 无重复文件/重复内容 | 仅 1 个文件，无重复段落 |
| 6 | 无异常恢复/漏跑/重跑 | 历史记录连续，无断档 |

## 5. 触发方式

### 主触发链（优先）：cron 定时任务
```bash
# 每日 00:05 UTC 执行
5 0 * * * /home/moonlight/.openclaw/workspace/tools/agent-health-check
5 0 * * * cd /home/moonlight/.openclaw/workspace && python3 tools/policy-daily-report --save
```

### 兜底触发：手动指令
- Telegram 发送：`继续 B2-3` 或 `B2-3 Day N`

**原则**：不依赖人工提醒作为长期方案

## 6. B0-5 触发条件

若出现以下任一情况，触发 B0-5：

| 条件 | 说明 |
|------|------|
| 连续 2 天失败 | 同一任务连续失败 |
| 状态丢失 | 历史记录断档 |
| 累积资源泄漏 | 内存/磁盘持续增长 |
| 无法恢复 | 中断后无法继续 |
| 漏跑/重跑 | 步骤被跳过或重复执行 |
| 状态与文件不同步 | 报告内容与真源不一致 |
