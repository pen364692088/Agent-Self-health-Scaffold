## Audit Checklist

### A. 围绕真实问题
- [ ] 用户痛点是否清楚表达
- [ ] 解决方案是否针对真实痛点
- [ ] 价值主张是否可信

### B. 范围控制
- [ ] 是否有越权动作
- [ ] 是否有伪数据/伪反馈
- [ ] 是否有 scope 膨胀

### C. 结构清晰度
- [ ] README 是否完整（安装、使用、示例）
- [ ] FAQ 是否覆盖主要问题
- [ ] 架构文档是否清晰

### D. 可继续推进价值
- [ ] 文档是否足够详细以继续开发
- [ ] 是否有明显的缺失关键信息
- [ ] demo 脚本是否可执行

### E. 验收标准
- [ ] 所有 Required Outputs 已创建
- [ ] FAQ 至少 10 个问题
- [ ] Landing page 有明确 value proposition
- [ ] Demo 脚本可执行（dry-run 模式）
- [ ] 架构文档清晰

## Required Outputs
1. 审计报告 (PASS/FAIL/PARTIAL)
2. 问题列表
3. 最小返工单（如有问题）
4. 改进建议（可选）

## Stop Conditions
- 完成所有 Audit Checklist 项目
- 输出审计结论
