# LEVELS.md - 自愈修复级别定义

## 修复级别总览

| 级别 | 名称 | 描述 | 自动执行 |
|-----|------|------|---------|
| L0 | 观察 | 仅记录、报警、人工审查 | ❌ |
| L1 | 沙箱修复 | 在临时环境执行修复并验证 | ✅ 沙箱内 |
| L2 | 修复分支提交 | 创建修复分支，产出 PR/报告 | ✅ 分支级 |
| L3 | 受限自动合入 | 通过所有 Gate 后自动合并 | ⚠️ 需配置 |

---

## L0 - 观察模式

### 触发条件
- 问题签名未识别
- 修复置信度低于阈值
- 涉及保护区边界

### 行为
1. 记录完整失败上下文
2. 生成 Failure Bundle
3. 发送告警通知
4. 等待人工审查

### 产物
- `failure_bundle_<id>.json`
- 告警消息

---

## L1 - 沙箱修复

### 触发条件
- 问题签名已识别
- 有历史成功修复记录
- 置信度 ≥ 60%

### 行为
1. 创建沙箱环境
2. 执行候选修复
3. 运行验证测试
4. 产出修复报告

### 约束
- 隔离执行环境
- 不影响主分支
- 可随时回滚

### 产物
- 沙箱修复日志
- 验证测试结果
- 修复报告

---

## L2 - 修复分支提交

### 触发条件
- L1 验证通过
- Gate A/B/C 全部通过
- 置信度 ≥ 80%

### 行为
1. 创建修复分支
2. 提交修复代码
3. 生成 PR 或报告
4. 通知审查者

### 约束
- 分支命名规范: `fix/<signature>-<timestamp>`
- 必须包含测试
- 必须有变更说明

### 产物
- 修复分支
- PR / 修复报告
- 测试更新

---

## L3 - 受限自动合入

### 触发条件
- L2 审查通过
- 所有 Gate 验证通过
- 置信度 ≥ 95%
- **显式配置允许**

### 行为
1. 合并到目标分支
2. 触发部署（如配置）
3. 更新经验库

### 约束
- 必须有回滚方案
- 必须有监控告警
- 必须有人工确认配置

### 产物
- 合并提交
- 部署记录
- 经验库更新

---

## 级别升降规则

### 升级条件
- 连续 5 次同签名成功修复 → 级别 +1
- 置信度达标 → 允许升级

### 降级条件
- 修复失败 → 级别 -1
- 引入新问题 → 级别 -2
- 人工干预 → 重置为 L0

### 级别上限
- 默认最高 L2
- L3 需显式配置开启

---

## Gate 验证对应关系

| 级别 | Gate A | Gate B | Gate C |
|-----|--------|--------|--------|
| L0 | - | - | - |
| L1 | ✅ | ✅ | ✅ |
| L2 | ✅ | ✅ | ✅ |
| L3 | ✅ | ✅ | ✅ |
