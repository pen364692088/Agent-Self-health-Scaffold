# SELF_HEALING_CONTRACT.md

## 自愈系统核心契约

### 1. 设计目标

将 Agent-Self-health-Scaffold 从"报错后补救"升级为：
- 失败自动接管
- 候选修复生成
- 沙箱验证
- 经验沉淀
- 预防复发
- 影子学习

**正确目标定义**：
> "同类问题被签名化、规则化、测试化，进入可识别、可拦截、可回归验证的闭环，复发率持续下降。"

**禁止误解**：
> 禁止把目标定义成"以后永远不会再出现这种问题"。

---

### 2. 失败接管范围

系统统一接收以下失败类型：

| 失败类型 | 说明 | 示例 |
|---------|------|------|
| `edit_failed` | 文件编辑失败 | edit 工具返回错误 |
| `tool_failed` | 工具执行失败 | exec 命令返回非零 |
| `test_failed` | 测试失败 | pytest 断言失败 |
| `build_failed` | 构建失败 | npm build 失败 |
| `runtime_failed` | 运行时失败 | 服务崩溃、异常退出 |

---

### 3. 修复流程

```
失败发生
    ↓
Failure Orchestrator 接管
    ↓
生成 Failure Bundle
    ↓
诊断归因 (L1分类 + L2签名)
    ↓
Remedy Planner 生成候选方案
    ↓
策略评分器排序
    ↓
Sandbox Healer 沙箱验证
    ↓
Gate A/B/C 验证
    ↓
[通过] 产出 Patch + Report
[失败] 回滚 + 人工升级
```

---

### 4. 强制边界

#### 4.1 禁止自动修改

以下区域**绝对禁止**自动修复：

- 核心身份 / 系统提示 (SOUL.md, IDENTITY.md)
- 安全策略 (POLICIES/)
- 凭据 / 密钥 (.env, secrets/)
- 主调度器核心 (subtask-orchestrate, callback-worker)
- Gate 验证器本身 (verify-and-close, done-guard)
- 权限白名单
- 自愈系统自己的边界约束 (本文件)

#### 4.2 执行约束

1. **沙箱优先**：所有自动修复默认只能在沙箱/临时分支/worktree执行
2. **Gate 强制**：未通过 Gate A/B/C 的方案禁止进入正式采纳
3. **外部学习隔离**：外部学习结果禁止直接修改正式策略库
4. **自动级别限制**：默认最高自动级别限制在"自动生成修复分支与报告"，禁止直接合主链

---

### 5. 修复级别定义

| 级别 | 名称 | 描述 | 自动执行 |
|-----|------|------|---------|
| L0 | 观察 | 仅记录、报警、人工审查 | ❌ |
| L1 | 沙箱修复 | 在临时环境执行修复并验证 | ✅ 沙箱内 |
| L2 | 修复分支提交 | 创建修复分支，产出 PR/报告 | ✅ 分支级 |
| L3 | 受限自动合入 | 通过所有 Gate 后自动合并 | ⚠️ 需配置 |

---

### 6. Gate 验证链

| Gate | 名称 | 验证内容 | 范围 |
|-----|------|---------|------|
| Gate A | Contract | 变更符合契约、不碰禁区 | 强制 |
| Gate B | E2E | 端到端测试通过 | self_healing 模块 scoped |
| Gate C | Preflight | tool_doctor 健康检查 | 强制 |

**Gate B 范围说明**:
- 由于项目测试结构存在同名文件冲突（test_execution_policy.py 在多处存在）
- Gate B 仅验证 self_healing 模块的 E2E 测试
- 其他模块的测试问题不属于自愈系统验收范围

---

### 7. 经验沉淀要求

系统必须保存四类记忆：

1. **问题记忆**：signature、触发条件、场景、影响范围
2. **成功修复记忆**：方案、成功前提、验证结果、成本
3. **失败修复记忆**：哪些修法误修、哪些条件下不能用
4. **预防规则**：edit 前 preflight 检查、高危目录保护

---

### 8. 影子学习约束

外部学习必须经过三段式：

1. **采集**：仅限官方文档、issue/PR、成熟框架
2. **蒸馏**：转成 pattern card / anti-pattern card / remediation template
3. **影子验证**：只进 shadow planner，通过观察窗口后才晋升

---

### 9. 验收标准

#### P0 (首期必须完成)
- [ ] `edit failed` 能自动接管
- [ ] failure bundle 生成成功
- [ ] 至少 3 个候选修法
- [ ] 可在沙箱执行验证
- [ ] 可产出 patch + report

#### P1 (经验沉淀)
- [ ] 同类问题第二次出现时能命中历史经验
- [ ] 能在 edit 前拦截已知高危问题
- [ ] 能自动补充回归测试

#### P2 (影子进化)
- [ ] 新知识默认只影响 shadow planner
- [ ] 有观察窗口和晋升机制
- [ ] 有数据证明复发率下降

---

### 10. 回滚与禁用

系统必须提供：

- **紧急禁用开关**：一键关闭所有自动修复
- **按签名禁用**：针对特定问题签名暂停自动修复
- **按级别降级**：L3→L2→L1→L0 逐级降级
- **完整回滚**：恢复到修复前状态
