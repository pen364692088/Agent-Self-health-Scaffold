# BOUNDARIES.md - 自愈系统边界约束

## 1. 绝对保护区 (禁止自动修改)

以下区域自愈系统**绝对禁止**触碰：

### 1.1 身份层
- `SOUL.md` - 核心身份
- `IDENTITY.md` - 角色定义
- `AGENTS.md` - Agent 行为准则

### 1.2 安全层
- `POLICIES/` - 安全策略目录
- `.env`, `secrets/` - 凭据与密钥
- 权限白名单文件

### 1.3 调度核心
- `subtask-orchestrate` - 主调度器
- `callback-worker` - 回调处理器
- Gate 验证器 (`verify-and-close`, `done-guard`)

### 1.4 自愈边界本身
- `contracts/BOUNDARIES.md` (本文件)
- `contracts/SELF_HEALING_CONTRACT.md`
- `contracts/LEVELS.md`

---

## 2. 受限操作区 (需显式授权)

以下区域需要显式授权才能修改：

### 2.1 工具层
- `tools/` 目录下的核心工具
- Gate 相关脚本

### 2.2 配置层
- `config/` 目录
- 状态文件 (`SESSION-STATE.md`, `TASK_STATE.md`)

### 2.3 测试层
- `tests/` 目录下的测试文件
- 测试配置

---

## 3. 可自动修复区

以下区域允许自动修复（仍需 Gate 验证）：

### 3.1 业务逻辑
- 非核心业务代码
- 工具辅助脚本

### 3.2 文档
- 非策略性文档
- 变更日志、报告

### 3.3 临时产物
- `artifacts/` 临时输出
- `reports/` 报告文件

---

## 4. 边界检查规则

### 4.1 前置检查
任何修复操作执行前必须：
1. 检查目标路径是否在保护区
2. 检查是否在受限区（需授权）
3. 记录边界检查结果

### 4.2 违规处理
- 保护区违规 → 立即中止 + 升级人工
- 受限区无授权 → 中止 + 请求授权
- 可修复区 → 继续 Gate 验证

---

## 5. 边界更新流程

边界文件本身只能通过以下方式更新：

1. **人工审查** - 必须有人工确认
2. **版本控制** - 必须有 commit 记录
3. **回滚能力** - 必须能回退到上一版本
